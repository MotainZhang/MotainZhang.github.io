web.xml配置详解

发表于 2017-05-08 | 分类于 Java Web ， Maven |

web.xml的作用

设置web应用程序的组件部署信息。

首页（欢迎页面）设置

可以指定多个<welcome-file>，按顺序加载。

Servlet声明

Servlet对外映射的路径是相对路径，支持模糊匹配，支持多个路径对应同一个Servlet。

ServletMapping匹配规则

优先级从高到低排列

  1. 精确路径匹配，完全匹配
  2. 最长路径匹配，最长前缀匹配
  3. 扩展名匹配
  4. default servlet(映射路径为/) 或者放弃匹配找不到对应的Servlet，报错）

ServletConfig配置

每个Servlet支持设置多个<init-param>。

ServletContext配置

<context-param>是在各个具体的<servlet>元素之外的，同样可以设置多个<context-param>。

如果需要在Servlet容器启动时执行操作

通过<load-on-startup>元素设置一个值，改变Servlet的默认初始化时间。
当值 >= 0时，Servlet容器启动时就会加载该Servlet；
当值 < 0时，则在容器第一次请求该Servlet时，才加载ervlet。

如果多个Servlet都设置了该值，数值越小，优先级越高。

如果访问页面不存在（404），如何更友好地显示页面

<error-code>当错误的返回码为设置的值时，跳转到指定的页面位置<location>。更高级的做法，添加<exception-type>元素，用来捕获Java的异常类型，使展示页面更有针对性。
